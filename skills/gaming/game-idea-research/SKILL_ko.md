## 목표

시장 트렌드 기반으로 **3개의 차별화된 장르**에서 2D 픽셀아트 + Phaser JS 게임 아이디어를 도출합니다.
각 아이디어는 레퍼런스 게임의 핵심을 계승하되, 독창적인 트위스트와 현대적 메커니즘으로 혁신해야 합니다.

### 3단계 — 아이디어 생성 (핵심 단계)

각 아이디어를 생성할 때 아래 **아이디어 품질 체크리스트**를 통과해야 합니다:

```
✅ 레퍼런스 게임이 명확히 존재하는가? (2000년대 이후)
✅ 단순 복제가 아닌 명확한 혁신 트위스트가 있는가?
✅ 핵심 게임루프가 1문장으로 설명되는가?
✅ 브라우저에서 마우스/키보드만으로 플레이 가능한가?
✅ 픽셀아트 16x16~32x32 스프라이트로 표현 가능한가?
✅ 외부 라이브러리 없이 Phaser 내장 기능으로 구현 가능한가?
✅ 세션 1회가 5~15분 이내로 완결되는가?
```

**혁신 트위스트 아이디어 생성 방법:**

레퍼런스 게임의 핵심 메커니즘에 아래 중 하나를 결합합니다:
- **장르 크로스오버**: A 장르 + B 장르 핵심 메커니즘 융합
- **시점/컨트롤 반전**: 적 시점, 역방향 진행, 환경 조작
- **자원 딜레마**: 기존 자원계를 제로섬 / 트레이드오프 구조로
- **시간 메커니즘**: 타임리밋, 리와인드, 예측 기반 플레이
- **정보 비대칭**: 안개, 기억 기반, 부분 공개 맵

---

### 4단계 — 아이디어 문서화

**모든 내용은 한국어 마크다운으로 작성합니다.**

각 아이디어의 필수 필드:

```markdown
## 아이디어 N: [제목]

### 기본 정보
- **id**: "kebab-case-id"
- **title**: "이모지 + 캐치한 제목"
- **description**: "한 줄 설명 (50자 이내)"
- **loader**: "() => import('./games/[id]/main.js')"

### 레퍼런스 게임
- **원작**: 게임명 (출시연도, 개발사)
- **장르**: 장르명
- **원작의 핵심**: 원작이 왜 성공했는지 1~2문장

### 혁신 트위스트
> 원작과 무엇이 다른가를 명확히 서술
- **트위스트 유형**: (장르 크로스오버 / 시점 반전 / 자원 딜레마 / 시간 메커니즘 / 정보 비대칭)
- **핵심 변경점**: 구체적인 메커니즘 차이 설명
- **플레이어 경험**: 플레이어가 어떤 감정/결정을 경험하는가

### 핵심 게임루프
```
[행동] → [결과] → [선택] → [반복]
예: 던전 탐색 → 자원 획득 → 업그레이드 선택 → 다음 층
```

### 픽셀아트 비주얼 가이드
- **해상도**: 16x16 / 32x32 스프라이트
- **팔레트**: 색상 수 제한 (예: 4~8색), 주요 톤 키워드
- **애니메이션**: 필요한 프레임 수, 핵심 애니메이션 목록
- **배경 스타일**: 타일맵 구성 방식

### Phaser JS 구현 포인트
- **Physics**: arcade / matter.js 중 선택 이유
- **Scene 구조**: 필요한 씬 목록 (Boot → Preload → Menu → Game → UI)
- **핵심 시스템**: 구현할 주요 클래스/모듈 목록
- **재사용 모듈**: 다른 게임과 공유 가능한 컴포넌트
- **난이도**: ⭐~⭐⭐⭐⭐⭐ (구현 복잡도)

### 시장성 평가
- **타겟 유저**: 주요 플레이어층
- **세션 길이**: 예상 1회 플레이 시간
- **재플레이성**: 낮음 / 중간 / 높음 (이유)
- **참고 성공 사례**: 유사 컨셉의 히트 인디게임
```

### 5단계 — 파일 저장

1. 오늘 날짜를 확인합니다:
   ```bash
   date +%Y%m%d
   ```

2. `idea/` 디렉토리가 없으면 생성합니다:
   ```bash
   mkdir -p idea
   ```

3. 파일을 저장합니다:
   ```bash
   # 파일명: idea/game_idea_YYYYMMDD.md
   ```

4. **반드시 `ls idea/`로 실제 파일명을 확인한 후** 다음 단계로 진행합니다:
   ```bash
   ls idea/game_idea_*.md
   ```

---

### 6단계 — Git 커밋 & 푸시

```bash
# 1. 실제 파일명 확인 (필수)
FILENAME=$(ls idea/game_idea_*.md | tail -1)
echo "커밋할 파일: $FILENAME"

# 2. Git add
git add "$FILENAME"

# 3. 커밋 (날짜 기반 메시지)
DATE=$(date +%Y%m%d)
git commit -m "feat: ${DATE} 게임 아이디어 추가"

# 4. 푸시
git push origin main
```

## 품질 기준

### ✅ 좋은 아이디어의 조건
- 핵심 루프가 5초 안에 설명됨
- "왜 지금 이 게임인가?"에 답할 수 있음
- 픽셀아트 스타일이 게임플레이와 유기적으로 연결됨
- Phaser 기본 기능(Arcade Physics, Tilemap, Tween, Group)만으로 구현 가능

### ❌ 피해야 할 아이디어
- 기존 게임의 단순 복제 (메커니즘 혁신 없음)
- 3D, 복잡한 물리, 대규모 AI가 필요한 게임
- 첫 세션에서 10분 이상 걸리는 튜토리얼이 필요한 게임
- 대량의 커스텀 에셋이 없으면 작동 안 되는 게임

## 주의사항 및 함정

| 상황 | 대처법 |
|------|--------|
| 검색 결과가 에셋팩/튜토리얼 위주 | itch.io 직접 탐색으로 즉시 전환 |
| 아이디어 3개가 비슷한 장르로 수렴 | 장르 선택 기준 재검토, 플레이 패턴 다양성 확인 |
| 파일명 불일치 | **항상 `ls`로 실제 파일명 확인 후 git add** |
| 혁신 트위스트가 막연함 | "트위스트 유형" 5가지 중 하나를 명시적으로 선택 |
| Phaser 구현이 과도하게 복잡 | 외부 라이브러리 없이 내장 기능만 사용하도록 축소 |
| 세 아이디어의 난이도가 동일 | 구현 난이도를 ⭐~⭐⭐⭐⭐⭐로 분산 |
